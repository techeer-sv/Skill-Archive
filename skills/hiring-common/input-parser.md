# 입력 파싱 가이드

> 모든 채용 스킬에서 공통으로 사용하는 입력 수집 및 파싱 절차.

---

## 입력 유형

### 1. 이력서 (Resume) - 필수

**지원 형식:**
- PDF 파일 (`.pdf`) — Read 도구의 pages 파라미터 활용
- Markdown 파일 (`.md`)
- 텍스트 파일 (`.txt`)

> DOCX, Notion URL 등은 직접 파싱이 불가능합니다.
> 해당 형식의 경우 PDF 또는 텍스트로 변환 후 제공하도록 안내하세요.

**파싱 절차:**
1. 사용자에게 이력서 파일 경로를 요청한다
2. `Read` 도구로 파일 내용을 읽는다
3. PDF의 경우 pages 파라미터로 분할 읽기 (10페이지 초과 시)
4. 읽기 실패 시 "PDF 또는 텍스트 형식으로 변환하여 다시 제공해주세요"로 안내

**이력서 미제공 시:**
```
이력서는 필수 입력입니다.
이력서 파일 경로를 알려주세요.

지원 형식: PDF, Markdown, TXT
(DOCX는 PDF로 변환 후 제공해주세요)
```

### 2. 포트폴리오 (Portfolio) - 선택

**지원 형식:**
- PDF 파일 (`.pdf`)
- Markdown 파일 (`.md`)
- 프로젝트 디렉토리 경로

**파싱 절차:**
1. 이력서 수집 후, 포트폴리오 제공 여부를 묻는다
2. 파일인 경우 `Read` 도구로 읽는다
3. 디렉토리인 경우 `Glob`과 `Read`로 주요 파일(README, 주요 소스 코드)을 탐색한다

### 3. 채용 공고 (Job Posting) - 스킬에 따라 필수/선택

**수집 방식 (우선순위):**
1. **직접 붙여넣기 (권장)**: "공고 내용을 직접 붙여넣어주세요"
2. **URL 제공**: WebFetch로 시도하되, 실패 시 직접 붙여넣기로 폴백

> 대부분의 채용 공고 사이트(원티드, 잡코리아, 사람인 등)는 JavaScript 렌더링이 필요하여
> URL로 직접 내용을 가져오기 어렵습니다. 직접 붙여넣기를 기본으로 안내하세요.

**추출 항목:**
- 회사명, 포지션명, 경력 요건
- 필수/우대 자격 요건
- 기술 스택 요구사항
- 업무 내용 / 역할
- 회사 문화 및 가치 (있는 경우)

---

## 입력 수집 흐름

각 스킬은 다음 순서로 입력을 수집한다:

```
1단계: 이력서 수집
  → "리뷰할 이력서 파일 경로를 알려주세요."
  → 미제공 시 진행 불가 안내

2단계: 포트폴리오 수집 (해당 스킬에서 필요한 경우)
  → "포트폴리오가 있으면 파일 경로를 알려주세요. (없으면 '없음')"
  → 미제공 시 포트폴리오 없이 진행

3단계: 채용 공고 수집
  → "채용 공고 내용을 붙여넣어주세요. 또는 URL을 알려주세요."
  → URL 제공 시 WebFetch 시도 → 실패 시 "직접 붙여넣어주세요" 안내
  → 미제공 시 일반적 기준으로 평가 진행

4단계: 추가 옵션 수집 (스킬별)
  → 난이도, 면접 유형 등 스킬별 옵션
```

## 입력 검증 및 에러 처리

| 상황 | 처리 |
|------|------|
| 파일 경로가 존재하지 않음 | "파일을 찾을 수 없습니다. 경로를 다시 확인해주세요." |
| Read 도구로 읽기 실패 | "이 파일 형식을 읽을 수 없습니다. PDF 또는 텍스트로 변환해주세요." |
| PDF가 10페이지 초과 | pages 파라미터로 분할 읽기 (1-10, 11-20 등) |
| URL 접근 실패 | "URL에서 내용을 가져올 수 없습니다. 공고 내용을 직접 붙여넣어주세요." |
| 이력서가 너무 짧음 (5줄 미만) | "이력서 내용이 너무 짧습니다. 올바른 파일인지 확인해주세요." |
