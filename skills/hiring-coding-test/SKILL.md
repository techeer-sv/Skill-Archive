# 코딩 테스트 시뮬레이션 (Hiring Coding Test)

포지션과 경력 수준에 맞는 코딩 테스트를 출제하고, 풀이를 평가합니다.

---

## 고유 평가 체계

> 코딩 테스트는 3축 평가(문제 인식/해결 과정/성과)를 직접 적용하지 않는다.
> 대신 아래 고유 점수 체계를 사용한다.

### 판정 기준
```
[PASS]       - 총점 70점 이상
[NEEDS_WORK] - 총점 40~69점
[REJECT]     - 총점 40점 미만
```

### 알고리즘 평가 항목

| 항목 | 비중 | 설명 |
|------|------|------|
| 정확성 | 30% | 올바른 결과를 반환하는가 |
| 시간 복잡도 | 20% | 최적 알고리즘을 사용했는가 |
| 공간 복잡도 | 10% | 메모리 효율성 |
| 코드 품질 | 20% | 가독성, 네이밍, 구조 |
| 엣지 케이스 | 10% | 경계 조건 처리 |
| 문제 분해 능력 | 10% | 문제를 어떻게 쪼개서 접근했는가 |

### 시스템 설계 평가 항목

| 항목 | 비중 | 설명 |
|------|------|------|
| 요구사항 분석 | 20% | 문제를 올바르게 이해하고 명확화 |
| 아키텍처 설계 | 25% | 전체 구조의 적합성 |
| 트레이드오프 인식 | 20% | 설계 결정의 장단점 인식 |
| 확장성 | 15% | 스케일링 전략 |
| 디테일 | 10% | 핵심 컴포넌트 세부 설계 |
| 커뮤니케이션 | 10% | 설명의 명확성 |

### 실무 코딩 평가 항목

| 항목 | 비중 | 설명 |
|------|------|------|
| 기능 구현 | 30% | 요구사항 충족 |
| 코드 품질 | 25% | 구조, 가독성, 유지보수성 |
| 에러 처리 | 15% | 예외 상황 대응 |
| 설계 결정 | 20% | 패턴 선택의 합리성 |
| 테스트 | 10% | 테스트 작성 여부 및 품질 |

---

## 실행 절차

### Step 1: 입력 수집

**포지션 정보 (필수):**
- "코딩 테스트를 준비할 포지션 정보를 알려주세요. (이력서 파일 경로 또는 포지션/기술스택 직접 입력)"
- 이력서 제공 시: Read로 읽어서 포지션/기술 스택/경력 수준 추출
- 직접 입력 시: 포지션명, 경력 수준, 기술 스택 확인

**채용 공고 (선택):**
- "채용 공고가 있으면 내용을 붙여넣거나 URL을 알려주세요. (없으면 '없음')"

**테스트 옵션:**
- 유형: "알고리즘/자료구조", "시스템 설계", "실무 코딩", "자동 선택"
- 난이도: "쉬움", "보통", "어려움", "자동 (포지션 기반)"

### Step 2: 문제 생성

포지션, 기술 스택, 난이도를 기반으로 문제를 생성한다.

#### 알고리즘 문제 형식
```markdown
## 코딩 테스트

**유형:** 알고리즘
**난이도:** [선택 난이도]
**참고 시간:** [30분~60분] (자기 관리. 별도 타이머를 설정하세요.)
**사용 언어:** 자유

---

### 문제: [문제 제목]

[문제 설명]

**입력:** [형식]
**출력:** [형식]
**예시:** [2~3개]
**제약 조건:** [범위, 시간/공간 제약]
```

#### 시스템 설계 문제 형식
```markdown
## 시스템 설계 과제

**포지션 레벨:** [Junior/Mid/Senior]
**참고 시간:** 45~60분

### 문제: [시스템명] 설계

[요구사항]

**기능 요구사항:** [목록]
**비기능 요구사항:** 트래픽, 가용성, 지연 시간
**설계 범위:** [포함/제외 범위]
```

#### 실무 코딩 과제 형식
```markdown
## 실무 코딩 과제

**기술 스택:** [포지션 관련]
**참고 시간:** 60~90분

### 과제: [제목]
[배경 및 요구사항]
```

### Step 3: 풀이 수신

문제 출제 후 풀이를 요청한다:
- "풀이를 제출해주세요. 코드를 직접 입력하거나 파일 경로를 알려주세요."
- 힌트 요청 시: 단계적으로 제공 (전체 답을 바로 주지 않음)
  - 1차 힌트: 접근 방향만
  - 2차 힌트: 핵심 알고리즘/패턴
  - 3차 힌트: 의사코드 수준

### Step 4: 풀이 평가

유형에 맞는 평가 항목별로 점수를 산출한다.

**문제 분해 능력 평가 (모든 유형 공통):**
- 문제를 어떻게 쪼개서 접근했는가?
- 핵심 제약 조건을 파악했는가?
- 접근법 선택의 논리적 근거가 있는가?

### Step 5: 결과 출력

```markdown
# 코딩 테스트 결과

## 종합 점수: [점수]/100 — [PASS / NEEDS_WORK / REJECT]

### 항목별 점수
| 평가 항목 | 점수 | 코멘트 |
|----------|------|--------|
| [항목] | [점수]/[만점] | [코멘트] |

### 풀이 분석

#### 접근법 평가
[사용자의 문제 해결 접근법 분석]

#### 코드/설계 리뷰
[구체적 피드백]

#### 문제 분해 능력
[문제를 어떻게 쪼개서 접근했는지에 대한 평가]

### 개선 포인트
1. [구체적 개선점과 방법]

### 모범 답안

#### 접근법 설명
[최적 접근법의 사고 과정]

#### 코드/설계
[모범 답안]

#### 복잡도 분석
- 시간: O(...)
- 공간: O(...)
```

## 주의사항

- 문제는 실제 기업 코딩 테스트 수준으로 출제한다
- 포지션/경력에 맞는 적정 난이도를 유지한다
- 모범 답안은 반드시 사용자 풀이 제출 후에 공개한다
- "참고 시간"은 실제 면접 참고용 정보일 뿐, Claude가 시간을 측정하지 않는다
