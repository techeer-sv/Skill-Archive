# 이력서 리뷰 (Hiring Resume Review)

채용 공고와 이력서를 분석하여 서류 통과 가능성을 평가하고 구체적인 개선 피드백을 제공합니다.

---

## 핵심 판정 기준 (인라인)

```
[PASS]       - 3축 모두 B 이상, 1개 이상 A
[NEEDS_WORK] - 3축 중 1개 이상 C, 또는 전반적으로 B이나 A 없음
[REJECT]     - 3축 중 1개 이상 D, 또는 2개 이상 C

3축: 문제 인식 / 해결 과정 / 성과·임팩트 (각 A/B/C/D)
등급 판정 시 이력서 원문 인용 필수.
```

---

## 실행 절차

### Step 0: 평가 프레임워크 로딩

다음 파일을 Read 도구로 읽고 내용을 숙지한 후 진행한다 (프로젝트 내 또는 `~/.claude/skills/` 경로):
- `hiring-common/evaluation-framework.md` — 3축 평가 체계, 등급 기준, 경력 수준별 캘리브레이션
- `hiring-common/input-parser.md` — 입력 파싱 및 에러 처리 가이드 (선택)

> 파일을 읽을 수 없는 경우, 위의 "핵심 판정 기준"만으로도 진행 가능하다.

### Step 1: 입력 수집

**이력서 (필수):**
사용자에게 이력서 파일 경로를 요청한다.
- "리뷰할 이력서 파일 경로를 알려주세요. (PDF, Markdown, TXT)"
- 미제공 시: "이력서는 필수입니다. 파일 경로를 알려주세요." 출력 후 대기
- DOCX 제공 시: "DOCX는 직접 읽을 수 없습니다. PDF로 변환 후 다시 제공해주세요."

**채용 공고 (선택):**
- "채용 공고가 있으면 내용을 붙여넣어주세요. URL도 가능합니다. (없으면 '없음')"
- URL 제공 시: WebFetch로 시도. 실패하면 "내용을 직접 붙여넣어주세요."
- 미제공 시: 일반적 기준으로 평가 진행 (JD 매칭 분석 생략)

### Step 2: 이력서 파싱

`Read` 도구로 이력서를 읽는다.
- PDF의 경우 pages 파라미터 활용 (10페이지 초과 시 분할)
- 읽기 실패 시: "파일을 읽을 수 없습니다. 다른 형식(PDF, TXT)으로 제공해주세요."

이력서에서 다음을 식별한다:
- 경력 요약 (Summary)
- 프로젝트/경험 목록
- 기술 스택
- 경력 수준 추정 (주니어/미드/시니어)

### Step 3: 채용 공고 분석 (제공 시)

공고에서 다음을 추출한다:
- 회사명, 포지션명, 경력 요건
- 필수/우대 자격 요건
- 기술 스택 요구사항
- 업무 내용 / 역할

### Step 4: 평가 수행

#### 4-1. 경력 요약 명확성
- 핵심 역량과 포지션의 연결이 드러나는가?
- 차별화 포인트가 있는가?

#### 4-2. 프로젝트별 3축 평가

각 프로젝트/경험에 대해 3축을 평가한다. 이력서 특성상 **"존재 여부"** 중심으로 평가한다:

**문제 인식 (축 1):**
- 프로젝트가 해결하려는 문제가 서술되어 있는가?
- "왜 이 문제가 중요한가"가 드러나는가?
- 이력서에 문제 정의가 없으면 → "근거 불충분"으로 표기 (부정적 추측 금지)

**해결 과정 (축 2):**
- 기술적 접근법이 서술되어 있는가?
- 의사결정 근거가 있는가? (대안 비교, 트레이드오프)
- 이력서에 과정 설명이 없으면 → "해결 과정에 대한 기술이 없어 평가 불가. 추가 기술 권장."

**성과 (축 3):**
- 정량적 성과가 있는가? (수치, 비율, 측정 기준)
- 성과 수치의 맥락이 있는가? (baseline 대비)
- 본인 기여와 팀 기여가 구분되는가?

> **중요**: 이력서에 트레이드오프, ROI, 대안 비교 정보가 없는 것이 일반적이다.
> 이를 "C등급"으로 판정하지 말고, "이 정보를 추가하면 더 강력해진다"는 개선 제안으로 전환하라.

#### 4-3. JD 매칭 분석 (공고 제공 시)
- 필수 요건 충족률 (각 항목별 O/X/△)
- 우대 요건 충족률
- 기술 스택 매칭률
- 누락된 핵심 키워드

#### 4-4. 이력서 전체 정량 점검
- 정량적 성과가 포함된 항목 수 / 전체 항목 수
- 성과의 신뢰도 분포

### Step 5: 결과 출력

다음 형식으로 출력한다:

```markdown
# 이력서 리뷰 결과

## 종합 판정: [PASS / NEEDS_WORK / REJECT]
경력 수준 기준: [주니어/미드/시니어]

### 평가 요약
| 평가 축 | 등급 | 핵심 근거 |
|---------|------|----------|
| 문제 인식 | [A/B/C/D] | "[이력서 원문 인용]" — [분석] |
| 해결 과정 | [A/B/C/D] | "[이력서 원문 인용]" — [분석] |
| 성과/임팩트 | [A/B/C/D] | "[이력서 원문 인용]" — [분석] |

### JD 매칭 (공고 제공 시)
| 구분 | 충족률 | 상세 |
|------|--------|------|
| 필수 요건 | X/Y | [각 항목 O/X/△] |
| 우대 요건 | X/Y | ... |
| 기술 스택 | X/Y | ... |

### 프로젝트별 상세 분석
[각 프로젝트의 문제 인식 → 해결 과정 → 성과 분석. 원문 인용 포함.]

### 강점
- [강점]: "[근거 인용]"

### 개선 필요 사항
- [개선점]: [구체적 수정 방법. 가능하면 수정 전/후 예시 제시]

### 이력서 수정 제안
[항목별 구체적 문구 수정 제안. "~를 ~로 변경" 수준의 구체성]

### 신뢰도
| 주요 주장 | 신뢰도 | 판단 근거 |
|----------|--------|----------|
| "[인용]" | 근거 충분/부분적/불충분 | ... |
```

## 주의사항

- 개인 정보(연락처, 주소)는 평가하지 않는다
- 학력 자체를 평가하지 않는다 (경험과 역량 중심)
- 추측하지 않는다. 확인 불가 항목은 "평가 불가"로 표기한다
- 이력서에 정보가 부족한 것은 C/D 등급이 아니라 "추가 기술 권장"으로 안내한다
