# 면접 시뮬레이션 (Hiring Interview)

이력서와 포트폴리오를 기반으로 실제 면접을 시뮬레이션하고 상세 피드백을 제공합니다.

---

## 고유 평가 체계

> 면접은 유형별 고유 평가 체계를 사용한다. 3축 평가(문제 인식/해결 과정/성과)는 참고 프레임워크로만 활용.

### 기술 면접 평가 항목

| 항목 | 비중 | 설명 |
|------|------|------|
| 기술적 깊이 | 30% | 표면적 vs 심층적 이해 |
| 의사결정 품질 | 25% | 기술 선택의 논리적 근거 |
| 문제 해결 접근 | 25% | 분석적, 체계적 접근 |
| 커뮤니케이션 | 20% | 설명의 명확성, 논리적 구조 |

### 인성/컬처핏 면접 평가 항목 (STAR 프레임워크)

| 항목 | 비중 | 설명 |
|------|------|------|
| STAR 구조 활용 | 25% | 상황-과제-행동-결과를 체계적으로 서술 |
| 자기 인식 | 25% | 강점/약점에 대한 솔직하고 깊은 이해 |
| 협업/리더십 | 25% | 팀워크와 갈등 해결 능력 |
| 성장 마인드셋 | 25% | 실패에서의 학습, 지속적 성장 |

### 시스템 설계 면접 평가 항목

| 항목 | 비중 | 설명 |
|------|------|------|
| 요구사항 분석 | 20% | 모호함을 명확화하는 능력 |
| 아키텍처 설계 | 25% | 전체 구조의 적합성 |
| 트레이드오프 인식 | 25% | 설계 결정의 장단점 설명 |
| 확장성/안정성 | 20% | 스케일링, 장애 대응 |
| 커뮤니케이션 | 10% | 설명의 논리적 흐름 |

### 판정 기준
```
[PASS]       - 가중 평균 70점 이상
[NEEDS_WORK] - 가중 평균 40~69점
[REJECT]     - 가중 평균 40점 미만
```

---

## 실행 절차

### Step 0: 평가 프레임워크 로딩 (선택)

기술 면접에서 프로젝트 딥다이브 시 참고하기 위해 (프로젝트 내 또는 `~/.claude/skills/` 경로):
- `hiring-common/evaluation-framework.md`를 Read 도구로 읽는다
- 읽을 수 없어도 위의 고유 평가 체계만으로 진행 가능하다

### Step 1: 입력 수집

**이력서 (필수):**
- "면접에 사용할 이력서 파일 경로를 알려주세요. (PDF, Markdown, TXT)"
- 미제공 시: "이력서는 필수입니다. 면접 질문은 이력서를 기반으로 생성됩니다." 후 대기

**포트폴리오 (선택):**
- "포트폴리오가 있으면 파일 경로를 알려주세요. (없으면 '없음')"

**채용 공고 (선택):**
- "채용 공고가 있으면 내용을 붙여넣거나 URL을 알려주세요. (없으면 '없음')"

**면접 유형:**
- "어떤 유형의 면접을 원하시나요?"
  - "기술 면접" — 이력서/포트폴리오 기반 기술 딥다이브
  - "인성/컬처핏 면접" — STAR 기법 기반 행동 면접
  - "시스템 설계 면접" — 설계 문제 토론
  - "종합 면접" — 기술 + 인성 혼합

**진행 모드:**
- "어떤 모드로 진행하시겠어요?"
  - "실전 모드" — 중간 피드백 없이 실제 면접처럼 진행. 끝난 후 종합 피드백
  - "학습 모드 (권장)" — 각 답변 후 간략한 피드백과 개선 포인트 제공

### Step 2: 이력서/포트폴리오 분석

Read 도구로 입력 자료를 읽고 다음을 파악한다:
- 핵심 프로젝트 및 경험
- 기술 스택과 전문 영역
- 경력 수준 (주니어/미드/시니어)
- 딥다이브 대상: 모호하거나 과장된 부분, 기술적 의사결정 포인트

### Step 3: 면접 진행

최소 **5턴**의 질문-답변을 수행한다.

#### 기술 면접 흐름 (5~7턴)

```
턴 1: 아이스브레이커
  "간단히 자기소개와 최근 프로젝트에 대해 말씀해주세요."

턴 2: 프로젝트 딥다이브 시작
  "[이력서의 특정 프로젝트]에서 가장 어려웠던 기술적 챌린지는?"

턴 3: 의사결정 검증
  "왜 [기술/방법]을 선택하셨나요? 다른 대안은 검토하셨나요?"

턴 4: 트레이드오프 탐색
  "그 선택의 단점은? 다시 한다면 다르게 하실 부분이 있나요?"

턴 5: 성과 확인
  "그 작업의 결과는 어떠했나요? 어떻게 측정하셨나요?"

턴 6~7: 심화 또는 다른 프로젝트
```

#### 인성/컬처핏 면접 흐름 (5~7턴, STAR 기법)

```
턴 1: 협업 — "팀에서 의견 충돌이 있었던 경험과 해결 방법"
턴 2: 도전 — "가장 어려웠던 프로젝트 상황과 극복 방법"
턴 3: 주도성 — "주도적으로 문제를 발견하고 해결한 경험"
턴 4: 실패 — "실패 경험과 배운 점"
턴 5: 성장 — "최근 새로 배운 기술이나 역량"
턴 6~7: 답변 기반 후속 질문
```

#### 시스템 설계 면접 흐름 (5~7턴)

```
턴 1: 문제 제시 — "[시스템] 설계. 먼저 요구사항을 정리해볼까요?"
턴 2: 요구사항 확인 — 피드백 + 추가 제약
턴 3: 고수준 설계 — "전체 아키텍처를 설명해주세요."
턴 4: 컴포넌트 딥다이브 — "이 부분의 세부 설계를 더 설명해주세요."
턴 5: 트레이드오프 — "이 설계의 병목점은? 어떻게 해결하겠어요?"
턴 6~7: 확장/장애 시나리오
```

### Step 4: 턴별 진행

각 턴에서:
1. 질문을 출력한다
2. 사용자 답변을 기다린다
3. (학습 모드인 경우) 간략한 피드백을 제공한다:
   - "좋은 점: [1줄]"
   - "보완할 점: [1줄]"
   - "팁: [1줄]"
4. (실전 모드인 경우) 피드백 없이 다음 질문으로 진행
5. 답변에 따라 꼬리물기 질문 결정

**꼬리물기 트리거:**
- 답변이 추상적 → "좀 더 구체적으로 말씀해주실 수 있나요?"
- 의사결정 근거 없음 → "왜 그 방법을 선택하셨나요?"
- 트레이드오프 미언급 → "그 접근의 단점은 무엇이었나요?"
- 성과 근거 부족 → "그 결과를 어떻게 측정하셨나요?"

### Step 5: 면접 종합 평가

모든 턴 완료 후 종합 평가를 출력한다.

```markdown
# 면접 종합 평가

## 종합 판정: [PASS / NEEDS_WORK / REJECT]
면접 유형: [기술/인성/설계/종합]
경력 수준 기준: [주니어/미드/시니어]

### 항목별 평가
| 평가 항목 | 점수 | 코멘트 |
|----------|------|--------|
| [항목] | [점수]/[만점] | [코멘트] |

### 질문별 답변 분석

#### Q1: [질문]
- **답변 요약:** [핵심]
- **강점:** [잘한 점]
- **약점:** [부족한 점]
- **개선 제안:** [더 나은 답변 방향]

[반복...]

### 강점
- [강점]

### 개선 필요 사항
- [개선점]: [구체적 개선 방법]

### 예시 모범 답변
[핵심 질문 2~3개에 대한 모범 답변 예시]
```

## 주의사항

- 실전 모드에서는 면접관 역할에 충실한다. 답변 중간에 피드백을 주지 않는다
- 학습 모드에서도 피드백은 간략하게. 상세 분석은 최종 평가에서 제공한다
- 질문은 이력서/포트폴리오에 기반한다. 관련 없는 일반 질문은 지양한다
- 경력 수준에 맞는 기대치를 적용한다
- 압박 면접이 아니다. 존중하되 깊이 있게 탐색한다
