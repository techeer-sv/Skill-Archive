# 면접 시뮬레이션 (Hiring Interview)

이력서와 포트폴리오를 기반으로 실제 면접을 시뮬레이션하고, 구조화된 루브릭으로 평가합니다.

---

## 판정 기준 (인라인)

```
기술 면접: 6단계 스케일 (Strong Hire ~ Strong Reject)
  [PASS]   — Leaning Hire 이상
  [REJECT] — Leaning Reject 이하

인성 면접: 5개 역량 영역, 각 1~5점
  [PASS]   — 5개 영역 모두 2점 이상 AND 평균 3점 이상
  [REJECT] — 어느 하나라도 1점 OR 평균 3점 미만
```

---

## 실행 절차

### Step 1: 입력 수집

**이력서 (필수):**
- "면접에 사용할 이력서 파일 경로를 알려주세요. (PDF, Markdown, TXT)"
- 미제공 시: "이력서는 필수입니다." 후 대기

**포트폴리오 (선택):**
- "포트폴리오가 있으면 파일 경로를 알려주세요. (없으면 '없음')"

**채용 공고 (선택):**
- "채용 공고가 있으면 내용을 붙여넣거나 URL을 알려주세요. (없으면 '없음')"

**면접 유형:**
- "어떤 유형의 면접을 원하시나요?"
  - "기술 면접" — 이력서/포트폴리오 기반 기술 딥다이브 + 시스템 설계
  - "인성/컬처핏 면접" — STAR 기법 기반 행동 면접
  - "종합 면접" — 기술 + 인성 혼합

**진행 모드:**
- "어떤 모드로 진행하시겠어요?"
  - "실전 모드" — 중간 피드백 없이 실제 면접처럼 진행. 끝난 후 종합 피드백
  - "학습 모드 (권장)" — 각 답변 후 간략한 피드백과 개선 포인트 제공

### Step 2: 이력서/포트폴리오 분석

Read 도구로 입력 자료를 읽고 다음을 파악한다:
- 핵심 프로젝트 및 경험
- 기술 스택과 전문 영역
- 경력 수준 (주니어/미드/시니어)
- 딥다이브 대상: 모호하거나 과장된 부분, 기술적 의사결정 포인트

### Step 3: 면접 진행

최소 **5턴**의 질문-답변을 수행한다.

#### 기술 면접 흐름 (5~7턴)

```
턴 1: 자기소개 + 최근 프로젝트 개요
  "간단히 자기소개와 최근 프로젝트에 대해 말씀해주세요."

턴 2: 프로젝트 딥다이브
  "[이력서의 특정 프로젝트]에서 가장 어려웠던 기술적 챌린지는?"

턴 3: 의사결정 검증
  "왜 [기술/방법]을 선택하셨나요? 다른 대안은 검토하셨나요?"

턴 4: 트레이드오프 / 시스템 설계
  "그 선택의 단점은? 다시 한다면 다르게 하실 부분?" 또는 시스템 설계 질문

턴 5: 성과와 측정
  "그 작업의 결과는? 어떻게 측정하셨나요?"

턴 6~7: 심화 또는 CS 기초
```

#### 인성/컬처핏 면접 흐름 (5~7턴, STAR 기법)

```
턴 1: 협업 — "팀에서 의견 충돌이 있었던 경험과 해결 방법"
턴 2: 주도성 — "주도적으로 문제를 발견하고 해결한 경험"
턴 3: 도전/실패 — "가장 어려웠던 상황과 극복 방법" 또는 "실패와 배운 점"
턴 4: 성장 — "최근 새로 배운 기술이나 역량, 학습 방법"
턴 5: 커뮤니케이션 — "복잡한 기술 내용을 비전문가에게 설명한 경험"
턴 6~7: 답변 기반 후속 질문
```

### Step 4: 턴별 진행

각 턴에서:
1. 질문을 출력한다
2. 사용자 답변을 기다린다
3. (학습 모드인 경우) 간략한 피드백:
   - "좋은 점: [1줄]"
   - "보완할 점: [1줄]"
   - "팁: [1줄]"
4. (실전 모드인 경우) 피드백 없이 다음 질문으로 진행
5. 답변에 따라 꼬리물기 질문 결정

**꼬리물기 트리거:**
- 답변이 추상적 → "좀 더 구체적으로 말씀해주실 수 있나요?"
- 의사결정 근거 없음 → "왜 그 방법을 선택하셨나요?"
- 트레이드오프 미언급 → "그 접근의 단점은 무엇이었나요?"
- 성과 근거 부족 → "그 결과를 어떻게 측정하셨나요?"
- STAR 불완전 (인성) → "그래서 결과적으로 어떻게 되었나요?" (Result 유도)

### Step 5: 면접 종합 평가

#### 기술 면접 평가

4개 차원별로 평가 후 종합 판정:

| 차원 | 비중 | 평가 내용 |
|------|------|----------|
| 기술적 깊이 | 30% | 기술 선택 근거, 내부 동작 이해, 트레이드오프 인식 |
| 문제 해결 | 25% | 체계적 접근, 문제 분해, 제약 조건 파악 |
| 시스템 설계 | 25% | 아키텍처 적합성, 확장성 (시니어 비중 ↑) |
| 커뮤니케이션 | 20% | 설명의 명확성, 논리적 구조 |

**6단계 판정:**
```
Strong Hire    — 반드시 채용. 모든 차원에서 기대 이상
Hire           — 채용 추천. 대부분 차원에서 우수
Leaning Hire   — 약간의 우려가 있지만 채용 가능
Leaning Reject — 우려가 있어 채용 추천 어려움
Reject         — 채용하지 않음
Strong Reject  — 명백히 부적합
```

#### 인성/컬처핏 면접 평가

5개 역량 영역별 1~5점:

| 영역 | 평가 포인트 |
|------|-----------|
| 협업/팀워크 | 의견 충돌 해결, 피드백 수용 |
| 주도성/오너십 | 자발적 문제 발견, 역할 범위 초월 |
| 성장 마인드셋 | 실패에서의 학습, 지속적 성장 |
| 커뮤니케이션 | 복잡한 상황의 명확한 설명 |
| 문화 적합도 | 회사 가치와의 정렬 (JD 기반) |

### Step 6: 결과 출력

```markdown

면접 유형: [기술/인성/종합]

### (기술 면접) 차원별 평가
| 차원 | 평가 | 코멘트 |
|------|------|--------|
| 기술적 깊이 | [상세] | [코멘트] |
| 문제 해결 | [상세] | [코멘트] |
| 시스템 설계 | [상세] | [코멘트] |
| 커뮤니케이션 | [상세] | [코멘트] |

**Hire 판정: [Strong Hire ~ Strong Reject]**

### (인성 면접) 역량별 평가

|------|-----------|--------|
| 협업/팀워크 | [점수] | [코멘트] |
| 주도성/오너십 | [점수] | [코멘트] |

**평균: [점수] / 5.0**

### 질문별 답변 분석

- **답변 요약:** [핵심]
- **강점:** [잘한 점]
- **약점:** [부족한 점]
- **개선 제안:** [더 나은 답변 방향]

[반복...]

### 개선 필요 사항
- [개선점]: [구체적 개선 방법]

[핵심 질문 2~3개에 대한 모범 답변 예시]
```

## 주의사항

- 실전 모드에서는 면접관 역할에 충실한다. 답변 중간에 피드백을 주지 않는다
- 학습 모드에서도 피드백은 간략하게. 상세 분석은 최종 평가에서 제공한다
- 질문은 이력서/포트폴리오에 기반한다. 관련 없는 일반 질문은 지양한다
- 경력 수준에 맞는 기대치를 적용한다
- 압박 면접이 아니다. 존중하되 깊이 있게 탐색한다
